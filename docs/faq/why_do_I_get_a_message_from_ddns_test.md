# DDNSテストでメッセージが表示されるのはなぜですか？

ダイナミックDNSページでDDNSテストを実行すると、以下のメッセージが表示される場合があります。

**「DDNSドメイン解決のIPアドレスがデバイスのWAN IPと同じではありません。」**

**「ダイナミックDNSを使用するには、互联网へのパブリックIPアドレスが必要です。」**

![ddnstest](https://static.gl-inet.com/docs/router/en/4/faq/warning_on_ddns_test/ddnstest.jpg){class="glboxshadow"}

これは**警告**や**エラーメッセージ**ではなく、ルーターのネットワーク状態を示すリマインダーです。

この結果は通例、ルーターのネットワーク位置を反映しています。例えば、GL.iNetルーターがホームネットワークでセカンダリールーターとして設定されている場合、このメッセージが表示されます。

プライマリールーターでポートフォワーディングを設定しても消えません。これは単にルーターがNATの下にあることを示しています。

NAT経由でサービス公開する必要がある場合（例：リモートアクセス用）は、追加の設定が必要です。
