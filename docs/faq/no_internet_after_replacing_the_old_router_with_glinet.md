# GL.iNetルーターに置き換えた後にインターネットがない場合の対処方法

古いルーターをGL.iNetルーターに置き換えた後、インターネットにアクセスできない場合、新しいルーターがWANポートがモデムに接続されているときにIPアドレス（DHCPまたはPPPoE経由）を取得できない可能性があります。

インターネットサービスプロバイダーが**MACバインディング**を有効にしている可能性があります。これは、以前のルーターのMACアドレスのみがネットワークへの接続を許可します。

この問題を解決するには、次の方法を試してください。

- 古いルーターのMACアドレスを新しいGL.iNetルーターにクローンします。
- モデムを再起動します。問題が持続する場合は、30分間電源を切って 다시 켜주세요.
- インターネットサービスプロバイダーに連絡してモデムをリセットしてください。
