# Bloqué sur "Rebooting" pendant le redémarrage du firmware

Si votre routeur reste bloqué sur "Rebooting" pendant un redémarrage du firmware, suivez les étapes ci-dessous pour résoudre le problème.

![stuck on rebooting](https://static.gl-inet.com/docs/router/en/4/faq/troubleshooting/stuck_on_rebooting/rebooting.png){class="glboxshadow"}

Après chaque étape, vérifiez si le problème est résolu. Si c'est le cas, ignorez les étapes restantes.

1. Attendez au moins 5 minutes pour voir si le processus de redémarrage se termine de lui-même. Le processus de redémarrage peut parfois prendre plus de temps que prévu, surtout après une mise à jour du firmware.

2. Si le processus de redémarrage ne se termine pas après 5 minutes, éteignez votre routeur en débranchant le câble d'alimentation. Attendez 30 secondes, puis rebranchez le câble d'alimentation et attendez que le routeur démarre.

3. Ouvrez un navigateur Web, accédez à `192.168.8.1` et connectez-vous. Si l'opération de redémarrage a réussi, vous serez redirigé vers le guide de configuration.

4. Activez le mode privé/incognito de votre navigateur et accédez à `192.168.8.1`. Si le processus de redémarrage a réussi, vous serez redirigé vers le guide de configuration.

5. Utilisez un autre navigateur (par exemple, Chrome, Edge ou Safari) pour accéder à `192.168.8.1`. Si le processus de redémarrage a réussi, vous serez redirigé vers le guide de configuration.

6. Si vous ne parvenez toujours pas à accéder au panneau d'administration Web du routeur ou si le routeur ne fonctionne pas correctement, essayez de réinitialiser le routeur à ses paramètres d'usine en suivant [ce guide](repair_network_or_reset_firmware.md/#reset-to-factory).
