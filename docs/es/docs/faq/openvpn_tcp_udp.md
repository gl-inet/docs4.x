# OpenVPN, TCP frente a UDP

El servidor OpenVPN del router GL.iNet admite tanto el protocolo TCP como UDP. ¿Cuál es la diferencia?

TCP es más fiable, pero más lento. UDP es más rápido, pero menos fiable.

Si necesita baja latencia y transmisión continua de datos para que una aplicación funcione correctamente, UDP es una mejor opción. En cambio, TCP es un protocolo fiable para transferir datos sin pérdidas durante la transmisión.

UDP es más adecuado para juegos, streaming o servicios VoIP. TCP es más adecuado para correo electrónico, navegación web y transferencia de archivos.

Recomendamos probar primero el protocolo UDP y cambiar a TCP solo si experimenta inestabilidad en la conexión, pérdida de paquetes u otros problemas relacionados con la fiabilidad.
