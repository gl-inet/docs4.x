# Firmware v4.9

このリリースでは、より精密なネットワーク制御、改善されたトラフィック管理、強化されたネットワークセキュリティ、刷新されたユーザーインターフェースに重点を置き、全体的なユーザー体験の向上を目的としています。[Firmware Download Center](https://dl.gl-inet.com/){target="_blank"}

## Flow Control

Flow Control は、ネットワークトラフィックの識別、監視、制御、フィルタリングを精密に行うための中核的なネットワーク管理モジュールです。ネットワークリソースの割り当てを最適化し、帯域幅の混雑を抑え、ネットワークアクセス動作を標準化することで、よりスムーズで安全、かつ制御しやすいネットワーク体験を提供します。Firmware v4.9 では、このモジュールが複数の実用的な機能と連携し、包括的なトラフィック管理を実現します。

Flow Control モジュールには、次の機能が含まれます。

!!! note "DPI Engine"
    
    Deep Packet Inspection 技術により、アプリケーションプロトコル、トラフィック種別、ネットワーク動作を正確に識別します。

!!! note "データ統計"
    
    リアルタイムおよび履歴のネットワークトラフィックデータを収集、分析、可視化し、ネットワーク状態を直感的に監視できます。

!!! note "コンテンツフィルタリング"
    
    不適切なネットワークコンテンツをインテリジェントに遮断およびフィルタリングし、ネットワークアクセスを標準化します。

!!! note "QoS (Quality of Service)"
    
    主要なアプリケーションのネットワーク品質を確保するため、帯域幅の割り当てとトラフィック優先度のスケジューリングを行います。

!!! note "SQM (Smart Queue Management)"
    
    ネットワークキューのスケジューリングを最適化し、遅延とパケットロスを低減して、よりスムーズなネットワーク通信を実現します。

!!! note "ペアレンタルコントロール"
    
    以前は **Applications** メニューに分類されていたこの機能は、v4.9 で **Flow Control** メニューに移動されました。アップグレードされた DPI Engine を活用して不適切なアプリケーションやネットワークコンテンツを正確に識別してブロックし、より専門的で精密なトラフィックベースのアクセス制限を実現します。

## VPN

Firmware v4.9 では、VPN モジュールの基盤となるルーティングロジックと操作インターフェースが全面的に改善されています。潜在的なルーティング競合を修正し、設定ロジックを簡素化し、操作の直感性を高めています。
    
主な調整内容は次のとおりです。

!!! note "独立したVPNグループ化"

    各VPNトンネルは、グループ間フェイルオーバーを行わない独立したグループとして動作します。ネットワークトラフィックが特定のVPNグループに一致した場合、現在のトンネルに障害が発生しても他のVPNグループへ自動的に切り替わらないため、安定して予測可能なトラフィックルーティングが確保されます。

!!! note "グループ内プロファイルフェイルオーバー"
    
    1つのVPNトンネルグループに複数の設定プロファイルを含めることができます。ユーザーは同じグループ内で各プロファイルの優先順位をカスタマイズでき、単一のプロファイルに障害が発生した場合でも、自動的な内部フェイルオーバーによってVPN接続を維持できます。
    
!!! note "\"Not Use VPN\" ポリシーの削除"
    
    VPNポリシー設定で従来使用されていた "Not Use VPN" オプションは、v4.9 で削除されました。これにより冗長な設定項目がなくなり、複数のポリシー設定による複雑なルーティングロジックの競合を効果的に回避できます。
    
!!! note "VPN Dashboard の再設計"
        
    VPN Dashboard は、より直感的なレイアウトで全面的に再設計されました。トンネル状態、接続情報、設定項目がより明確に表示され、日常的な操作と管理の効率が大きく向上します。

## AmneziaWG 2.0 プロトコル

Firmware v4.9 では、複数の新しいトラフィック難読化パラメータを備えた AmneziaWG 2.0 プロトコルが正式に導入されました。アップグレードされたプロトコルは、DPI やその他のトラフィック識別システムによる検出を効果的に回避し、接続の秘匿性と耐干渉性を大幅に向上させます。これにより、ネットワーク制限のある地域や複雑なネットワーク環境でも、安定した信頼性の高いVPN接続を確立できます。

## IoTネットワーク

新しく構築されたIoTネットワーク機能では、IoTスマートデバイス専用の独立したWi-Fiネットワークを作成できます。メインネットワークから物理的および論理的に分離されるため、IoTデバイスがメインネットワークへアクセスすることによるネットワークリソースの消費やセキュリティリスクを回避できます。この最適化により、さまざまなスマートIoTクライアントとの互換性が広がり、ホームネットワーク全体のセキュリティが向上します。

## ACL (Access Control List)

ACL は Access Control List の略で、接続プロトコル、デバイスのIPアドレス、ポートに基づいて内部および外部ネットワークトラフィックを管理するカスタムアクセスルールを作成できる中核的なネットワークセキュリティ管理機能です。特定のネットワークアクセス動作を許可またはブロックするための精密な権限制御に対応しています。複数のACLルールが競合した場合、システムは優先度の高いルールを自動的に実行し、ポリシーが正確に適用されるようにします。

Port Forwarding との主な違いは役割です。ACL はデバイスとトラフィックのアクセス権限を制御するネットワークセキュリティ管理に重点を置きます。一方、Port Forwarding はネットワークリソースのリダイレクトに使用され、外部ネットワークトラフィックを指定したローカル端末へ転送して、ローカルネットワークサービスへのリモートアクセスを実現します。

## その他の改善

!!! note "Wireless UI の再設計"
    
    Wireless 設定インターフェースは、整理されたレイアウトと統一されたビジュアルスタイルで全面的に再設計されました。操作の複雑さが軽減され、インターフェース全体のシンプルさと使いやすさが大きく向上しています。

!!! note "暗号化DNSのアップグレード"
    
    暗号化DNSは、DoH、DoT、DoQ など、より多くの暗号化プロトコルに対応しました。同時に、より多くの公式DNSサービスプロバイダーが統合され、カスタム暗号化DNSサーバーの手動設定も追加されました。これにより、安全なドメイン解決に関する多様なニーズに対応できます。
    
!!! note "Tailscale Exit Node サポート"
    
    GL.iNetルーターは、Tailscale Exit Node として動作できるようになりました。Tailnet 内のデバイスからのすべての送信インターネットトラフィックをルーターのパブリックIPアドレス経由でルーティングでき、Tailscaleネットワーク全体に対して統一された安全なネットワーク出口管理を実現します。
    
!!! note "AstroWarp 統合"
    
    AstroWarp は v4.9 で GL.iNet Router SDK に正式に統合されました。ネイティブのトラフィック難読化機能を備えた AmneziaWG プロトコルを基盤とし、安定した暗号化済みの安全なリモートアクセスを提供します。ユーザーは Web Admin Panel 上の動的アクセスコードを使用して、デバイスのペアリングと接続設定をすばやく完了できます。アカウント登録やログインなしで、トラベルルーターとホームネットワーク間の安全なワンクリック接続を数秒で確立できます。
