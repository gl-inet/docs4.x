# ファームウェア再起動中に「Rebooting」でスタックする

ルーターがファームウェア再起動中に「Rebooting」でスタックする場合は、以下の手順に従って問題を解決してください。

![stuck on rebooting](https://static.gl-inet.com/docs/router/en/4/faq/troubleshooting/stuck_on_rebooting/rebooting.png){class="glboxshadow"}

各手順の後、問題が解決したかどうかを確認します。解決した場合は、残りの手順をスキップできます。

1. 再起動プロセスが自動的に完たするかどうかを確認するため、最も低5分待ちます。再起動プロセスは想定より長くかかることがあり、特にファームウェアアップデート後は。

2. 5分後も再起動プロセスが完たしない場合は、電源ケーブルを抜いてルーターの電源を切ります。30秒待してから、電源ケーブルを再度挿し込み、ルーターが起動するのを待ちます。

3. Webブラウザを開き、`192.168.8.1`にアクセスしてログインします。再起動操作が成功した場合は、セットアップガイドにリダイレクトされます。

4. ブラウザのプライベート/シークレットモードを有効にし、`192.168.8.1`にアクセスします。再起動プロセスが成功した場合、セットアップガイドにリダイレクトされます。

5. 別のブラウザ（Chrome、Edge、Safariなど）を使用して`192.168.8.1`にアクセスします。再起動プロセスが成功した場合、セットアップガイドにリダイレクトされます。

6. ルーターのWeb管理パネルにまだアクセスできない、またはルーターが正常に機能していない場合は、このガイドに従ってルーターを工場出荷時のデフォルト設定にリセットしてみてください。
