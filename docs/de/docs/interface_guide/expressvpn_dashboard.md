# ExpressVPN Dashboard

> **Note:** Diese Anleitung gilt nur für den GL.iNet x ExpressVPN Co-Branding-Router **Fortify**.

[ExpressVPN](https://www.expressvpn.com/){target="_blank"} ist einer der weltweit führenden Premium-VPN-Dienste zum Schutz Ihrer Online-Privatsphäre, zur Absicherung Ihrer Internetverbindung und zum Zugriff auf globale Inhalte. Der Dienst bietet sehr hohe Geschwindigkeiten, starke Verschlüsselung und Zugriff auf Server in über 100 Ländern. Ob Streaming, privates Surfen oder Schutz Ihrer Daten in öffentlichem Wi-Fi: ExpressVPN bietet ein schnelles und sicheres Nutzungserlebnis.

**Fortify** ist ein gemeinsam von GL.iNet und ExpressVPN veröffentlichter Co-Branding-Router. Jedes Gerät enthält ein kostenloses einjähriges ExpressVPN-Abonnement. Benutzer können das Abonnement direkt im web Admin Panel des Routers einlösen und ihre Konten dort binden. Nach der Aktivierung nutzt der gesamte Datenverkehr, der über den Router läuft, das Hochgeschwindigkeitsnetzwerk und die robuste Verschlüsselung von ExpressVPN, um Ihre gesamte Netzwerkverbindung und Online-Privatsphäre zu schützen.

Diese Anleitung führt Sie durch das Einlösen des 12-monatigen ExpressVPN-Tarifs im web Admin Panel des Routers. Außerdem wird erläutert, wie Sie VPN-Richtlinien an Ihre Nutzungsszenarien und Anforderungen anpassen, damit Sie verschlüsselte, sichere und schnelle Internetverbindungen nutzen können.

## VPN-Tunnel hinzufügen

Folgen Sie den nachstehenden Schritten, um Ihren VPN-Tunnel hinzuzufügen und die VPN-Richtlinien zu konfigurieren. Lesen Sie bei Bedarf die [Fallbeispiele](#fallbeispiele).

### Allgemeine Schritte

1. Melden Sie sich am web Admin Panel Ihres Fortify an und gehen Sie zu **VPN** -> **ExpressVPN Dashboard**. Klicken Sie auf **Add VPN Tunnel**.

    ![dashboard](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/1-dashboard_initial.png){class="glboxshadow"}

2. Wählen Sie ein VPN-Profil aus und klicken Sie anschließend auf **Next**.

    Sie erhalten eine Liste von ExpressVPN-Profilen. Wählen Sie ein oder mehrere Profile aus und passen Sie deren Priorität rechts nach Bedarf an.

    ![select profile](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/select-profile.png){class="glboxshadow"}

    !!! note

        Wenn mehrere Profile ausgewählt sind, versucht der Tunnel, die Verbindung in der Prioritätsreihenfolge der Profile herzustellen, bis eine Verbindung erfolgreich aufgebaut wurde. Wenn alle Profile innerhalb eines einzelnen Tunnels keine Verbindung herstellen können, entscheidet das System anhand des Status des Kill Switch in den Einstellungen [Tunneloptionen](#tunneloptionen) und [All Other Traffic](#all-other-traffic), ob zum lokalen Netzwerk (WAN) des Routers gewechselt wird.

3. Wählen Sie die Client-Quelle aus und klicken Sie anschließend auf **Next**.

    Es gibt vier Optionen:

    - **All Clients**: Wenn diese Option ausgewählt ist, entspricht der Datenverkehr von allen Geräten dieser Regel.
    ![all clients](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/all-clients.png){class="glboxshadow"}

    - **Specified Connection Types**: Wenn diese Option ausgewählt ist, entspricht der Datenverkehr von bestimmten Verbindungstypen (z. B. LAN-Subnetz, Drop-in Gateway, Gastnetzwerk) dieser Regel.
    ![specified connection types](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/specified-connection.png){class="glboxshadow"}

    - **Specified Devices**: Wenn diese Option ausgewählt ist, entspricht der Datenverkehr von bestimmten Geräten (identifiziert anhand der MAC-Adresse) dieser Regel.
    ![specified devices](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/specified-devices.png){class="glboxshadow"}

    - **Exclude Specified Devices**: Wenn diese Option ausgewählt ist, entspricht der Datenverkehr von bestimmten Geräten (identifiziert anhand der MAC-Adresse) nicht dieser Regel.
    ![exclude specified devices](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/exclude-specified-devices.png){class="glboxshadow"}

4. Wählen Sie das Ziel aus und klicken Sie anschließend auf **Apply**.

    Es gibt drei Optionen:

    - **All Targets**: Wenn diese Option ausgewählt ist, wird der Datenverkehr, der dieser Regel entspricht, zu allen Zielen geroutet.
    ![all targets](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/all-targets.png){class="glboxshadow"}

    - **Specified Domain / IP List**: Wenn diese Option ausgewählt ist, wird der Datenverkehr, der dieser Regel entspricht, zu bestimmten Domains oder IP-Adressen geroutet. Sie müssen diese manuell eingeben.
    ![specified domain ip](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/specified-domain-ip.png){class="glboxshadow"}

    - **Exclude specified Domain / IP List**: Wenn diese Option ausgewählt ist, wird der Datenverkehr, der dieser Regel entspricht, nicht zu bestimmten Domains oder IP-Adressen geroutet. Sie müssen diese manuell eingeben.
    ![exclude specified domain ip](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/exclude-specified-domain-ip.png){class="glboxshadow"}

5. Ein VPN-Tunnel wurde erfolgreich hinzugefügt. Sie werden zum **ExpressVPN Dashboard** weitergeleitet. Fügen Sie bei Bedarf weitere VPN-Tunnel hinzu.

### Fallbeispiele

Zwei typische Konfigurationsbeispiele mit schrittweisen Einrichtungsanweisungen stehen Ihnen als Referenz zur Verfügung.

**Fall 1**: Nur bestimmte Geräte über VPN routen.

<u>Anforderungen</u>:

1. Nur bestimmte mit diesem Router verbundene Geräte greifen über das VPN auf das Internet zu. Alle anderen Geräte greifen über das lokale WAN auf das Internet zu.

2. Die ausgewählten Geräte dürfen nur die VPN-Verbindung verwenden. Wenn die VPN-Verbindung unerwartet getrennt wird, wird der Internetzugang für diese Geräte blockiert, um DNS-Leaks und IP-Tracking zu verhindern.

<u>Konfigurationsschritte</u>:

1. Wählen Sie das VPN-Profil aus.

    Wählen Sie ein oder mehrere Profile aus, passen Sie deren Priorität rechts nach Bedarf an und klicken Sie dann auf **Next**.

    ![case 1 select profile](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-select-profiles.png){class="glboxshadow"}

2. Wählen Sie die Client-Quelle aus.

    Klicken Sie auf die Registerkarte **Specified Devices**, wählen Sie die Geräte aus, die das VPN verwenden sollen, und klicken Sie anschließend auf **Next**.

    ![case 1 source](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-specified-devices.png){class="glboxshadow"}

3. Wählen Sie das Ziel aus.

    Klicken Sie auf die Registerkarte **All Targets**, legen Sie sie als Datenverkehrsziel fest und klicken Sie anschließend auf **Apply**.

    ![case 1 target](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-all-targets.png){class="glboxshadow"}

4. Sie werden zum ExpressVPN Dashboard weitergeleitet. Ein VPN-Tunnel wurde nun erfolgreich hinzugefügt, wie unten gezeigt.

    ![case 1 dashboard](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-tunnel-apply.png){class="glboxshadow"}

5. Stellen Sie sicher, dass der **Kill Switch** für diesen Tunnel aktiviert ist. Wenn die VPN-Verbindung unerwartet getrennt wird, wird der Internetzugang für den Datenverkehr, der diesem Tunnel entspricht, blockiert, um DNS-Leaks und IP-Tracking zu verhindern.

    ![case 1 kill switch1](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-killswitch1.png){class="glboxshadow"}

    ![case 1 kill switch2](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-killswitch2.png){class="glboxshadow"}
    
6. Stellen Sie sicher, dass **Allow Non-VPN Traffic** aktiviert ist. Diese Option ist standardmäßig aktiviert, damit Datenverkehr, der nicht dem VPN-Tunnel entspricht, weiterhin über das lokale WAN-Netzwerk auf das Internet zugreifen kann.

    ![case 1 allow non-vpn traffic](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-allow-non-vpn.png){class="glboxshadow"}

7. Klicken Sie auf die mittlere Schaltfläche, um diesen Tunnel zu aktivieren.

    ![case 1 connect](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-start-vpn.png){class="glboxshadow"}

8. Sobald die Verbindung hergestellt ist, zeigt die Seite die VPN-Verbindungsdetails an, darunter VPN-Richtlinie, virtuelle Client-IP, Serveradresse, Listening-Port und Datenverkehrsstatistiken.

    ![case 1 connected](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/case1-connected.png){class="glboxshadow"}

    Nun greifen nur zwei bestimmte Geräte über VPN auf das Internet zu. Wenn die VPN-Verbindung unerwartet getrennt wird, wird der Internetzugang für diese Geräte blockiert, um DNS-Leaks und IP-Tracking zu verhindern. Alle anderen Geräte greifen stattdessen über das lokale WAN-Netzwerk auf das Internet zu.

## Kill Switch

Kill Switch ist eine Sicherheitsfunktion für VPN-Verbindungen. Sie trennt automatisch den gesamten Internetzugang Ihres lokalen Netzwerks, wenn die VPN-Verbindung unerwartet abbricht. Dadurch wird verhindert, dass Ihre echte IP-Adresse und Online-Daten offengelegt werden, und kontinuierliche Privatsphäre und Sicherheit werden gewährleistet. Diese Funktion ist besonders nützlich, um sicheren, anonymen Internetzugang aufrechtzuerhalten, etwa bei der Nutzung öffentlicher Netzwerke, der Verarbeitung sensibler Daten oder dem Verbergen Ihrer echten IP-Adresse.

Wenn die Funktion aktiviert ist, blockiert sie jeglichen Client-Datenverkehr, der versucht, den VPN-Tunnel zu umgehen, und verhindert so VPN-Leaks durch DNS-Konfigurationsprobleme, unerwartete Verbindungsabbrüche, direkte IP-Anfragen und ähnliche Szenarien.

Fortify unterstützt die Kill Switch-Konfiguration sowohl für die globale VPN-Verbindung als auch für jeden einzelnen VPN-Tunnel.

- Informationen zum Einrichten des Kill Switch für die globale VPN-Verbindung (d. h. Enhanced Kill Switch) finden Sie unter [All Other Traffic](#all-other-traffic).

- Informationen zum Einrichten des Kill Switch für jeden einzelnen VPN-Tunnel finden Sie unter [Tunneloptionen](#tunneloptionen).

## All Other Traffic

Klicken Sie oben rechts auf das Zahnradsymbol, um die Richtlinie für Datenverkehr einzurichten, der keinem VPN-Tunnel entspricht.

![all other traffic](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/all_other_traffic.png){class="glboxshadow"}

Diese Richtlinie steuert, ob Datenverkehr, der keiner Ihrer VPN-Tunnelgruppen entspricht, auf das Internet zugreifen kann. Es sind zwei Optionen verfügbar: **Allow Non-VPN Traffic** und **Enhanced Kill Switch**.

- **Allow Non-VPN Traffic**: Standardmäßig aktiviert, um normalen Internetzugang für Nicht-VPN-Datenverkehr sicherzustellen.

    ![allow non-vpn traffic](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/allow_non-vpn_traffic.png){class="glboxshadow"}

- **Enhanced Kill Switch**: Erzwingt, dass alle Geräte über ein VPN auf das Internet zugreifen. Jeglicher Datenverkehr, der keinem VPN-Tunnel entspricht, wird blockiert. Diese globale Einstellung überschreibt nicht den Kill Switch, der für einzelne VPN-Tunnel konfiguriert ist.

    ![enhanced killswitch](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/enhanced_killswitch.png){class="glboxshadow"}

## Tunneloptionen

Sie können erweiterte Einstellungen für jeden VPN-Tunnel konfigurieren, z. B. VPN Kill Switch, IP-Maskierung und MTU.

Klicken Sie in einer Tunnelgruppe auf das Zahnradsymbol und wählen Sie **Options**.

![tunnel options1](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/tunnel_options1.png){class="glboxshadow"}

![tunnel options2](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/tunnel_options2.png){class="glboxshadow"}

- **Kill Switch**: Wenn diese Option aktiviert ist, wird der Datenverkehr, der diesem VPN-Tunnel entspricht, blockiert, falls die VPN-Verbindung unerwartet fehlschlägt. Wenn sie deaktiviert ist, führt solcher Datenverkehr ein Failover zum Tunnel **All Other Traffic** aus.

- **Services from GL.iNet Use VPN**: Wenn diese Option aktiviert ist, übertragen GoodCloud-, DDNS- und rtty-Dienste Pakete über VPN-Tunnel. Diese Option ist standardmäßig deaktiviert, da diese Dienste normalerweise die echte IP-Adresse des Geräts benötigen, um ordnungsgemäß zu funktionieren.

- **Allow Remote Access to the LAN Subnet**: Wenn diese Option aktiviert ist, ist der Fernzugriff auf diesen Router und seine LAN-Geräte über das VPN erlaubt. Dazu muss der VPN-Server eine Route zurück zu seinem LAN-Subnetz ankündigen.

- **IP Masquerading**: Wenn diese Option aktiviert ist, werden die Quell-IP-Adressen der LAN-Clients in die VPN-Tunnel-IP des Routers umgeschrieben. Deaktivieren Sie diese Option nur für Site-to-Site-Konfigurationen, bei denen die Gegenstelle Ihre LAN-Subnetze kennt.

- **MTU**: Der für den Tunnel festgelegte MTU-Wert überschreibt die MTU-Einstellungen in der Konfigurationsdatei.

## Tunnelpriorität

Um die Tunnelpriorität anzupassen, klicken Sie in einer Tunnelgruppe auf das Zahnradsymbol und wählen Sie **Priority**.

![priority1](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/priority1.png){class="glboxshadow"}

Klicken und halten Sie das Drei-Linien-Symbol rechts, um die Tunnel neu anzuordnen, und klicken Sie anschließend auf **Apply**.

![priority2](https://static.gl-inet.com/docs/router/en/4/interface_guide/expressvpn_dashboard/priority2.png){class="glboxshadow"}

**Wenn mehrere Tunnel aktiviert sind, routet der Router den Datenverkehr nach den folgenden Regeln**:

1. Der Datenverkehr versucht zuerst, der Tunnelregel mit der höchsten Priorität zu entsprechen. Bei Übereinstimmung wird er über diesen Tunnel geroutet. Andernfalls wird die Tunnelregel mit der nächsthöheren Priorität geprüft und so weiter.

2. Jede Tunnelgruppe arbeitet unabhängig. Sobald Datenverkehr einer Tunnelregel entspricht, wird er über diesen Tunnel geroutet und führt kein Failover zwischen Tunnelgruppen aus.

3. Innerhalb jeder Tunnelgruppe können mehrere Profile ausgewählt werden, um ein Failover innerhalb des Tunnels zu ermöglichen. Wenn das Profil mit der höchsten Priorität in einer Tunnelgruppe ausfällt, verbindet sich der Tunnel automatisch mit dem Profil der zweithöchsten Priorität und so weiter.

4. Wenn ein VPN-Tunnel unerwartet getrennt wird, entscheidet das System anhand dessen, ob der **Kill Switch** dieses Tunnels aktiviert ist, ob der Datenverkehr zum Tunnel All Other Traffic wechseln darf.

    - Wenn der Kill Switch aktiviert ist, wird der Datenverkehr blockiert und führt kein Failover zum Tunnel All Other Traffic aus.
    - Wenn der Kill Switch deaktiviert ist, führt der Datenverkehr ein Failover zum Tunnel All Other Traffic aus.

5. Im Tunnel **All Other Traffic** bestimmen verschiedene Modi, ob Datenverkehr, der keinem VPN-Tunnel entspricht, auf das Internet zugreifen kann.

    - **Allow Non-VPN Traffic**: Standardmäßig aktiviert, damit Datenverkehr, der keinem VPN-Tunnel entspricht, weiterhin über das lokale WAN auf das Internet zugreifen kann.

    - **Enhanced Kill Switch**: Erzwingt, dass alle Geräte über ein VPN auf das Internet zugreifen. Jeglicher Datenverkehr, der keinem VPN-Tunnel entspricht, wird blockiert. Diese globale Einstellung überschreibt nicht den Kill Switch, der für einzelne VPN-Tunnel konfiguriert ist. Kurz gesagt, sie verstärkt den Kill Switch und blockiert regulären Internetzugang, um IP-Leaks zu verhindern.
