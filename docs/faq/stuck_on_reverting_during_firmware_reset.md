# ファームウェアリセット中に「Reverting」でスタックする

ルーターがファームウェアリセット中に「Reverting」でスタックする場合は、以下のStepsに従って問題を解決してください。

![stuck on reverting](https://static.gl-inet.com/docs/router/en/4/faq/troubleshooting/stuck_on_reverting/reverting.png){class="glboxshadow"}

各Stepの後、問題が解決したかどうかを確認します。解決した場合は、残りのStepをスキップできます。

1. リバーティングプロセスが自動的に完たするかどうかを確認するため、最も低10分待ちます。リバーシングプロセスは想定より長くがかかることがあります。

2. 10分後もリバーティングプロセスが完たしない場合は、電源ケーブルを抜いてルーターの電源を切ります。30秒待してから、電源ケーブルを再度挿し込み、ルーターが起動するのを待ちます。

3. Webブラウザを開き、`192.168.8.1`にアクセスしてログインします。リバート操作が成功した場合は、セットアップガイドにリダイレクトされます。

4. ブラウザのプライベート/シークレットモードを有効にし、`192.168.8.1`にアクセスします。リバーティングプロセスが成功した場合、セットアップガイドにリダイレクトされます。

5. 別のブラウザ（Chrome、Edge、Safariなど）を使用して`192.168.8.1`にアクセスします。リバーtingプロセスが成功した場合、セットアップガイドにリダイレクトされます。

6. ルーターのWeb管理パネルにまだアクセスできない、またはルーターが正常に機能していない場合は、このガイドに従ってルーターを工場出荷時のデフォルト設定にリセットしてみてください。
