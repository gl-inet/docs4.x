# IPv6

IPv6（Internet Protocol version 6）は、IPv4を置き換えるために設計された最も新のインターネットプロトコルです より大きな一意のIPプールを提供し、IPv4のアドレス枯渇問題 を解決し、世界のに増加する接続されたデバイスをサポートします。

ウェブ管理パネルの左側 -> ネットワーク -> IPv6。このページでは、ルーターでIPv6を有効にして設定できます。

![ipv6](https://static.gl-inet.com/docs/router/en/4/tutorials/ipv6/ipv6.png){class="glboxshadow"}

IPv6が有効になっていると、イーサネットなどのWANインターフェースはDHCPv6経由でIPv6アドレスを取なければならないします。イーサネット設定ページでIPv6アドレスを手動で変更することもできます。

**注意**: 一部の機能（例：ファイアウォール、GoodCloud、OpenVPN DCO）はまだIPv6をサポートしていません。これらの機能とIPv6を同時に使用すると、接続問題が発生する可能性があります。

**IPv6を有効にする**をオンにし、メinnネットワークのモードとDNS取なければならない方法を選択して、**適用**をクリックします。

![ipv6 enabled](https://static.gl-inet.com/docs/router/en/4/tutorials/ipv6/ipv6_enabled.png){class="glboxshadow"}

**モード**: 4つのモードが利用可能です：**Native**、**Passthrough**、**NAT6**、**Static IPv6**。

- Native: このモードは、ルーターが直接パブリックIPv6アドレスを取なければならないし、ルーターがオンラインデバイスにIPv6アドレスを自動的に割り当てるときに適用されます。このモードはほとんどのユーザーのIPv6アクセスニーズを満たすことができます。

- Passthrough: このモードは、IPv6パケットを処理や変換なしに直接通過させる必要があるときに適用されます。例えば、特定のネットワークアプリケーションやサービスでは、さらなる処理や分析のためにIPv6パケットのコンテンツを完全に保持する必要がある場合があり、これは技術者がネットワークのデバッグやセキュリティ分析に使用します。

- NAT6: このモードは、ルーターを管理ゲートウェイとして使用し、ネットワーク上の各デバイスに動のな内部IPv6アドレスを割り当てるシナリオに適しています。このモードでは、ターミナルデバイスは光ネットワークターミナル経由で接続し、ローカルエリアネットワークのIPv6アドレスを取なければならないします。

- Static IPv6: このモードは、サーバーやネットワークプリンターなど、固定IPv6アドレスを必要とするデバイスやサービスに適しています。このモードにより、デバイスは常に同じIPv6アドレスを使用するため、管理とアクセスが簡単になります。

**DNS取なければならない方法**: ルーターがIPv6 DNSサーバーアドレスを取なければならないする方法を決定します。2つのオプションがあります：**から動**と**手動**。

- から動: ルーターはIPv6 DNSサーバーアドレスを動のに取なければならないします（例：DHCPv6経より）。

- 手動: カスタムIPv6 DNSサーバーアドレスを入力します。ただし、DNSはドメイン名に対応するIPアドレスを解決するために使用されるため、手動のDNSサーバー設定はDNS検索の失敗につながる可能性があります。使用には注意してください。
