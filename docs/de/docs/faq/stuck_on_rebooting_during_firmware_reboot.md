# Hängt bei „Rebooting“ während des Firmware-Neustarts

Wenn Ihr Router während eines Firmware-Neustarts bei „Rebooting“ hängen bleibt, befolgen Sie die folgenden Schritte, um das Problem zu beheben.

![stuck on rebooting](https://static.gl-inet.com/docs/router/en/4/faq/troubleshooting/stuck_on_rebooting/rebooting.png){class="glboxshadow"}

Prüfen Sie nach jedem Schritt, ob das Problem behoben ist. Wenn ja, überspringen Sie die verbleibenden Schritte.

1. Warten Sie mindestens 5 Minuten, um zu prüfen, ob der Neustartvorgang von selbst abgeschlossen wird. Dieser Vorgang kann manchmal länger dauern als erwartet, insbesondere nach einer Firmware-Aktualisierung.

2. Wenn der Neustartvorgang nach 5 Minuten nicht abgeschlossen ist, schalten Sie Ihren Router aus, indem Sie das Stromkabel abziehen. Warten Sie 30 Sekunden, schließen Sie das Stromkabel wieder an und warten Sie, bis der Router hochgefahren ist.

3. Öffnen Sie einen Webbrowser, rufen Sie `192.168.8.1` auf und melden Sie sich an. Wenn der Neustart erfolgreich war, werden Sie zum Einrichtungsassistenten weitergeleitet.

4. Aktivieren Sie den privaten/Inkognito-Modus Ihres Browsers und rufen Sie `192.168.8.1` auf. Wenn der Neustart erfolgreich war, werden Sie zum Einrichtungsassistenten weitergeleitet.

5. Verwenden Sie einen anderen Browser (z. B. Chrome, Edge, Safari), um `192.168.8.1` aufzurufen. Wenn der Neustart erfolgreich war, werden Sie zum Einrichtungsassistenten weitergeleitet.

6. Wenn Sie weiterhin nicht auf das Web-Admin-Panel des Routers zugreifen können oder der Router nicht ordnungsgemäß funktioniert, versuchen Sie, den Router mithilfe [dieser Anleitung](repair_network_or_reset_firmware.md/#reset-to-factory) auf die Werkseinstellungen zurückzusetzen.
